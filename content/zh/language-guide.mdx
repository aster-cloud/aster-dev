# 语言指南

Aster 是一种面向规则的**受控自然语言**（CNL）。它读起来像英文（或中文 / Deutsch /
हिन्दी），却编译成精确、确定性的中间表示。本指南讲解核心构造。

## 语法

每个文件是一个**模块**，含一条或多条**规则**。语句以句号结尾。

```aster
Module Pricing.

Rule discountedPrice given amount as Int, produce Int:
  If amount greater than 100
    Return amount times 90 divided by 100.
  Return amount.
```

- `Module <名称>.` 声明模块。
- `Rule <名称> given <参数>:` 声明规则（函数）及其输入。
- 每条语句以句号结尾；规则返回最后一个 `Return` 的值。

## 类型

Aster 是带推断的静态类型。核心类型：整数、小数、文本、布尔、列表、记录。
你很少需要显式写类型——它们从字面量和字段访问推断而来。

## 表达式

算术、比较与逻辑都用词语表达：

```aster
Module Underwriting.

Rule eligible given age as Int, income as Int, produce Bool:
  Return age is at least 18 and income is greater than 30000.
```

## 规则与函数

规则把输入映射为决策。用 `If … Return …` 分支：第一个匹配的分支胜出，末尾的
`Return` 作为默认值：

```aster
Module Tiering.

Rule classify given amount as Int, produce Text:
  If amount greater than 10000
    Return "large".
  If amount greater than 1000
    Return "medium".
  Return "small".
```

## 模块

模块组织相关规则，并可按版本相互引用，因此决策总能钉住它运行时的确切规则集——
这是**可回放**决策的基石。
