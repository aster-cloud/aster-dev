# 什么是 Aster？

Aster 是一种面向业务规则的**受控自然语言**（CNL）。策略读起来像句子——却编译成
确定性的核心，可经 REST、GraphQL、WebSocket 或直接在浏览器中运行，每次求值都带可
回放的决策轨迹。

Aster 介于自然英语与传统编程语言之间：足够受限，机器能无歧义解析；又足够自然，业务
专家——核保员、合规官、产品经理——无需开发者协助即可阅读与编写。

一个最小策略：

```aster
模块 定价。

规则 折扣价格 给定 金额 作为 整数，等级 作为 文本，产出 整数：
  如果 等级 等于 "gold"
    返回 金额 乘以 80 除以 100。
  返回 金额。
```

## 核心特性

- **确定性** —— 相同输入总产生相同输出，无副作用。
- **类型化** —— 每个参数与返回值都带类型（`Int`/`Float`/`Text`/`Bool`/`DateTime`
  或自定义结构），尽量推断。
- **多语言** —— 用 English、中文、Deutsch 或 हिन्दी 编写；所有 locale 编译到同一核心。
- **可审计** —— 引擎为每次求值记录防篡改的决策轨迹。

## 如何组合

```
  编写 (CNL)  ──►  浏览器 API (aster-lang-ts)  ──►  REST API (aster-api)
                   校验 · schema · 样例              求值 · 版本 · 审计
```

1. **编写** —— 在[演练场](/playground)或任意编辑器写策略。
2. **校验** —— 部署前抓语法/类型错误（浏览器 API 或演练场）。
3. **部署** —— 把源码提交给 REST API；引擎编译并存储。
4. **求值** —— 用输入数据调用已部署策略；得到类型化结果 + 审计记录。
5. **监控** —— 查版本历史、审计日志、异常检测，保持合规。
