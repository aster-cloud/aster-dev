# Language Guide

Aster is a **controlled natural language** (CNL) for rules. It reads like English
(or 中文 / Deutsch / हिन्दी) but compiles to a precise, deterministic intermediate
representation. This guide walks the core constructs.

## Syntax

Every file is a **module** with one or more **rules**. Statements end with a period.

```aster
Module Pricing.

Rule discountedPrice given amount as Int, produce Int:
  If amount greater than 100
    Return amount times 90 divided by 100.
  Return amount.
```

- `Module <Name>.` declares the module.
- `Rule <name> given <params>:` declares a rule (function) and its inputs.
- Each statement ends with a period; the rule returns the value of its last
  `Return`.

## Types

Aster is statically typed with inference. The core types:

| Type | Example |
| --- | --- |
| Integer | `42` |
| Decimal | `3.14` |
| Text | `"approved"` |
| Boolean | `true`, `false` |
| List | `[1, 2, 3]` |
| Record | `applicant.creditScore` |

You rarely write types — they are inferred from literals and field access.

## Expressions

Arithmetic, comparison and logic read in words:

```aster
Module Underwriting.

Rule score given a as Int, b as Int, produce Int:
  Return a times 2 plus b.

Rule eligible given age as Int, income as Int, produce Bool:
  Return age is at least 18 and income is greater than 30000.
```

Comparisons (`is at least`, `is greater than`, `is equal to`) and logic
(`and`, `or`, `not`) keep the source scannable.

## Rules & functions

A rule maps inputs to a decision. Branch with `If … Return …`; the first matching
branch wins, and a trailing `Return` acts as the default:

```aster
Module Tiering.

Rule classify given amount as Int, produce Text:
  If amount greater than 10000
    Return "large".
  If amount greater than 1000
    Return "medium".
  Return "small".
```

## Modules

Modules group related rules and can reference each other by version, so a decision
always pins the exact rule set it ran against — the foundation of **replayable**
decisions.
